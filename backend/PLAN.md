# 后端开发计划（Go）

## 目标
以规格驱动方式实现后端能力，支撑博客、Labs 与打卡学习平台的前端需求，并为后续扩展留出空间。

## 里程碑拆解
- M0：需求对齐与接口草案确认
- M1：数据模型与数据库设计确认
- M2：项目脚手架与基础设施准备（配置、日志、迁移）
- M3：P0 接口交付（博客/ Labs 只读 + 打卡 CRUD）
- M4：统计与筛选能力（标签、搜索、打卡统计）
- M5：安全与权限（可选，JWT/角色）
- M6：测试、监控与部署流程

## 规格驱动流程（Spec-Driven）
1) 产出接口规格（路径、请求/响应、错误码）
2) 明确数据模型与字段校验规则
3) 定义验收标准（DoD）与边界场景
4) 设计最小可用 API 集合（P0）
5) 再扩展高级能力（P1）

## 交付物清单
- API 规格文档（`SPEC.md`）
- 项目说明与架构约定（`PROJECT.md`）
- 数据模型草案与字段说明
- 端到端验收清单（DoD）

## 风险与注意事项
- 博客内容以 MDX 为主，后端以“元数据读取”定位为主
- 打卡数据涉及长期存储与统计，需优先明确时间与时区规则
- 若引入登录体系，需确认是否单用户或多用户模式
