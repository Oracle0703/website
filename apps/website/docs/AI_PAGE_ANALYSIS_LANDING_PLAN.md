# AI 页面分析与改版方案助手 - 落地实现计划（v2）

## 目标
在 `apps/website` 将 `ai-page-analysis` 升级为 `作品展示型科技产品页面`，并提供单页端到端演示流：输入页面素材、观察阶段生成、查看结构化改版输出。

## 固定范围（必须实现）
- 保留并完成全部 7 个既定分区（Hero、痛点、工作流、Demo、价值、场景、Footer CTA）。
- Demo 输入支持 `URL / 截图说明 / 业务 Brief` 标签切换。
- Demo 中间态升级为阶段化流水线（解析、提取、诊断、方案生成）。
- Demo 结果升级为产品化输出（评分、问题、蓝图、效果展示、交付节奏、Backlog）。
- 视觉语言对齐科技产品页，不改全站主题机制。

## 实施步骤
1. 文档更新：将 Spec/Plan 从初版修订为 v2 固定方向。
2. Demo 架构升级：补全输入台 + 流水线状态机 + 结果台。
3. 输出内容升级：新增更完整的展示字段（分析编号、Before/After、交付节奏、Owner/ETA）。
4. 视觉强化：在现有 token 上增强控制台式展示观感。
5. 校验：运行 `npm run lint:website`（必要时补充 build）。

## 非目标
- 不接真实后端模型。
- 不新增账号、导出、计费或历史记录。
- 不改全站导航与跨页面 IA。

## 完成定义（DoD）
- 页面目标明确为作品展示，文案不再偏泛营销。
- 用户可在单页完成“输入 → 阶段生成 → 完整结果”体验。
- 所有既定分区均保留并可用。
- website lint/build 至少通过一项并可报告结果。
