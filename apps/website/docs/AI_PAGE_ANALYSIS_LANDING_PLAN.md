# AI 页面分析与改版方案助手 - 落地实现计划（初版）

## 目标
在 `apps/website` 交付一个可访问的初版 Landing Page，用于展示“网页分析（URL / 截图 / Brief）→ 结构化改版建议”的产品价值与体验路径。

## 范围（本次）
- 新增 1 个落地页路由（含 SEO metadata）。
- 页面包含：Hero、痛点、工作流、Demo 输入/输出、价值、适用场景、底部 CTA。
- Demo 为前端 Mock 交互，不接入真实模型。
- 产出实现计划与规格文档。

## 不在范围（本次）
- 不接入后端 API / 模型服务。
- 不做账号体系、历史记录、计费、导出能力。
- 不调整全站导航信息架构（仅新增页面与文档）。

## 实施步骤
1. 规划文档：输出实现计划与规格说明（IA、文案、交互、技术结构）。
2. 页面骨架：建立新路由，完成 Landing Page 分区布局。
3. Demo 交互：实现 URL / 截图 / Brief 三模式输入、Mock 生成状态与结构化结果展示。
4. 细节统一：复用现有 tokens / 组件风格（`panel-surface`、`card-interactive`、typography 常量）。
5. 可发现性：将新页面加入 `sitemap`。
6. 校验：运行 website lint 或 build，记录结果。

## 验收标准（DoD）
- 页面可访问且结构完整。
- Demo 可切换输入模式并产出 Mock 结果。
- 文案与布局传达“分析 + 改版建议”的核心价值。
- 代码改动最小且符合现有代码风格。
- 至少完成一次 website app 的 lint/build 检查。
