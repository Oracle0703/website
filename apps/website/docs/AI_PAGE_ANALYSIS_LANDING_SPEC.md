# AI 页面分析与改版方案助手 - Landing Spec（初版）

## 1. 信息架构（IA）
页面自上而下结构：
1. Hero：价值主张 + 主次 CTA + 能力标签。
2. 痛点区：当前网页改版常见问题（方向不清、协作低效、优先级混乱）。
3. 工作流：输入 → 诊断 → 方案输出。
4. Demo 区：左侧输入区（URL/截图/Brief），右侧输出区（结构化建议卡片）。
5. 价值区：相比传统方式的效率和决策收益。
6. 场景区：电商落地页、SaaS 官网、活动页、品牌官网等。
7. Footer CTA：引导预约试用/提交页面进行评估。

## 2. 内容与文案
### Hero
- 标题：`AI 页面分析与改版方案助手`
- 副标题：`输入 URL、截图或简要需求，快速得到可执行的改版建议。`
- 主 CTA：`立即体验 Demo`
- 次 CTA：`查看分析样例`

### 痛点文案
- `评估标准不统一：团队对“哪里有问题”缺少共识。`
- `改版优先级混乱：同时出现太多意见，无法排布执行顺序。`
- `输出难落地：建议停留在抽象描述，设计与开发无法直接执行。`

### 工作流文案
- `Step 1 输入页面素材`（URL / 截图 / Brief）
- `Step 2 生成诊断洞察`（结构、文案、视觉、转化路径）
- `Step 3 输出改版方案`（问题证据、优先级、改造动作、预期收益）

### 价值文案
- `减少无效讨论，统一评估语言。`
- `优先级清晰，先做高影响改动。`
- `从“建议”直接过渡到“执行任务”。`

## 3. Demo 交互规格（Mock）
### 输入模式
- 模式切换：`URL` / `截图说明` / `业务 Brief`。
- 每种模式有默认示例内容，可手动编辑。
- 点击 `生成分析建议` 后进入 loading（约 1 秒）再显示结果。

### 输出结构
- 健康度概览：`信息清晰度`、`视觉层级`、`转化路径` 三项评分。
- 核心问题列表：每项含 `严重级别`、`证据`、`影响`。
- 改版建议列表：每项含 `模块`、`动作`、`优先级`、`预期收益`。
- 可执行 Backlog：以任务条目形式展示（如 Hero、表单、信任背书、CTA）。

### 体验原则
- 不调用真实模型，结果由前端 Mock 数据驱动。
- 输出看起来真实、可执行、结构统一。
- 保持移动端与桌面端均可读可操作。

## 4. 技术结构（website app）
### 路由与文件
- `app/ai-page-analysis/page.tsx`：页面入口 + metadata。
- `components/landing/ai-page-analysis-landing-client.tsx`：Demo 交互与页面主内容。
- `app/sitemap.ts`：新增页面路径。
- `docs/AI_PAGE_ANALYSIS_LANDING_PLAN.md` 与 `docs/AI_PAGE_ANALYSIS_LANDING_SPEC.md`：项目文档。

### 实现约束
- 复用现有 UI 风格：`panel-surface`、`card-interactive`、主题色变量、typography 常量。
- 默认采用客户端状态管理（`useState`）驱动 demo。
- 不新增外部依赖，不改全局主题策略。

### SEO（初版）
- title：`AI 页面分析与改版方案助手`
- description：强调“输入页面素材，产出结构化改版建议”。
- canonical：`/ai-page-analysis`

## 5. 初版验收
- 页面结构完整且符合 IA。
- Demo 可输入、可切换模式、可触发 mock 输出。
- 样式与全站 baseline 一致，无明显风格割裂。
- lint/build 至少通过一项。
