# AI 页面分析与改版方案助手 - Landing Spec（v2）

## 0. 固定前提（Yu）
1. 页面目标固定为 `作品展示`，不是泛营销落地页。
2. 已规划分区全部为必选，不允许删除。
3. 视觉方向固定为 `科技产品页`。
4. Demo 必须体现可信的阶段化生成过程，不能“瞬时替换静态结果”。
5. 用户必须可在单页中体验完整闭环：输入 → 生成 → 输出。

## 1. 信息架构（全部必选）
页面自上而下保持 7 个分区：
1. Hero（定位为作品演示入口）
2. 痛点区（为什么需要分析与改版助手）
3. 工作流区（输入、流水线、输出）
4. Demo 区（输入台 + 生成流水线 + 结果台）
5. 价值区（对协作效率与执行质量的提升）
6. 场景区（可展示的业务场景）
7. Footer CTA（回到 Demo / 联系需求）

## 2. 交互规格（v2）
### 2.1 输入台
- 三种输入标签页：`URL` / `截图说明` / `业务 Brief`。
- 每种模式提供可编辑默认示例。
- CTA 固定为完整运行：`生成完整改版演示`。

### 2.2 生成流水线（阶段化）
- 流水线阶段固定为：
  1. `解析页面素材`
  2. `提取结构与层级`
  3. `诊断问题与影响`
  4. `生成改版方案`
- 每阶段依次推进，显示 `等待 / 进行中 / 完成`。
- 显示总体进度条与过程日志，构建“可信生成感”。

### 2.3 结果台（更像产品 Demo）
必须包含：
- 分析摘要（分析编号、时间、置信度）
- 健康度评分（至少 3 维）
- 核心问题（严重度 + 证据 + 影响）
- 改版蓝图（模块 + 动作 + 优先级 + 预期收益）
- 改版后预期效果展示（Before/After + 指标预估）
- 交付节奏（分阶段交付说明）
- 可执行 Backlog（任务 + Owner + ETA + Priority）

## 3. 视觉与体验方向
- 风格：科技产品展示页，强调“控制台/流水线/结果台”体验。
- 复用站点现有样式与 token（`panel-surface`、`card-interactive`、颜色变量、typography 常量）。
- 保持单页结构与移动端可读性。
- 仍为前端 Mock，不接入真实模型。

## 4. 技术实现
- `app/ai-page-analysis/page.tsx`：路由入口与 metadata。
- `components/landing/ai-page-analysis-landing-client.tsx`：页面主体、输入态、流水线状态机、结果展示。
- 文档同步更新：
  - `docs/AI_PAGE_ANALYSIS_LANDING_PLAN.md`
  - `docs/AI_PAGE_ANALYSIS_LANDING_SPEC.md`

## 5. 验收标准（v2）
- 7 个分区全部存在且顺序完整。
- Demo 可完成一次端到端体验（输入、阶段生成、结果输出）。
- 生成过程具备序列推进与日志反馈，不是瞬时静态替换。
- 页面视觉符合科技产品展示语义。
- 至少完成一次 website `lint` 或 `build` 校验并记录结果。
