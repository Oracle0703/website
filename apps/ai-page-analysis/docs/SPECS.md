# AI 页面需求分析助手 - Specs

## 产品定位
- 类型：公司内部可共享使用的独立产品
- 目标：给定 URL、页面截图或业务 Brief，输出研发可执行的需求解读
- 受众：产品、运营、设计、前端、后端、项目负责人

## 输入
1. URL
2. 页面截图 / 截图说明
3. 业务 Brief

## 输出
1. 需求类型判断
2. 页面目标解读
3. 产运需求拆解
4. 必须实现项
5. 接口建议
6. 关键实现点
7. PRD 缺失项
8. 研发拆解建议
9. `plan.md` 草稿
10. `spec.md` 草稿

## 页面结构
1. Hero
2. 痛点
3. 工作流
4. Demo 控制台
5. 输出结果区
6. 价值说明
7. 场景区
8. Footer CTA

## Demo 流程
1. 用户选择输入模式
2. 输入 URL / 截图说明 / Brief
3. 点击生成
4. 依次展示：识别页面类型 -> 拆解产运需求 -> 定位关键实现点 -> 生成研发解读
5. 最终输出结构化结果

## 视觉方向
- 科技产品页
- 深色基底
- 蓝青色强调色
- 控制台式输出区
- 结果形态更像“需求评审台”而不是营销页改版建议
