# AI 页面分析与改版方案助手 - Specs

## 产品定位
- 类型：公司内部可共享使用的独立产品
- 目标：理解网页输入，并输出可执行的改版建议
- 受众：产品、设计、运营、增长、售前、项目负责人

## 输入
1. URL
2. 截图说明
3. 业务 Brief

## 输出
1. 页面定位
2. 结构评分
3. 核心问题
4. 改版蓝图
5. 预期收益
6. 执行 Backlog

## 页面结构
1. Hero
2. 痛点
3. 工作流
4. Demo 控制台
5. 价值说明
6. 场景区
7. Footer CTA

## Demo 流程
1. 用户选择输入模式
2. 填写内容
3. 点击生成
4. 依次展示：解析 -> 提取 -> 诊断 -> 方案生成
5. 最终输出结构化结果

## 视觉方向
- 科技产品页
- 深色基底
- 蓝青色强调色
- 控制台式结果面板
- 强调“流程可信”和“结果可执行”

- `GET /api/healthz`
- `POST /api/demo`
