# 通用后台管理页面规格说明（Spec）

> 目标：在当前目录规划一个通用性较强的后台管理页面（不写实现代码），给出前后端技术选型、任务拆分与接口文档草案。页面简洁：左侧菜单栏，右侧内容区；右侧内容区上下分区，上部为面包屑 + 登录信息；下部为页面具体内容；点击登录信息可下拉并支持“退出登录”。

---

## 1. 需求概述

### 1.1 功能范围（包含但不限于）
- 登录 / 注册
- 用户管理
- 菜单与权限管理（RBAC）
- 首页统计展示（仪表盘）
- 通用布局（左侧菜单，右侧内容区）
- 右上角登录信息下拉（含退出登录）

### 1.2 非功能性需求
- 页面简洁、可扩展、易维护
- 支持多角色与权限
- 具备前后端分离、接口可复用
- 可在后期扩展更多业务模块

---

## 2. 前端技术选型（建议）

### 2.1 栈建议
- **框架**：React 18
- **语言**：TypeScript
- **构建工具**：Vite
- **路由**：React Router v6
- **状态管理**：Zustand（轻量）或 Redux Toolkit（复杂场景）
- **UI 组件库**：Ant Design（成熟后台组件）或 Arco Design（轻量现代）
- **样式**：Less / CSS Modules
- **图表**：ECharts
- **请求库**：Axios
- **权限控制**：前端路由守卫 + 菜单按权限渲染

### 2.2 选型理由简述
- React + Vite 组合上手快、社区活跃、可维护性强
- TS 强化类型安全，适合复杂管理系统
- Ant Design 适配后台模板与权限场景

---

## 3. 后端技术选型（建议）

### 3.1 栈建议（偏 Node 方案）
- **框架**：NestJS（结构化、可扩展）
- **语言**：TypeScript
- **数据库**：PostgreSQL / MySQL
- **ORM**：Prisma
- **认证**：JWT + Refresh Token
- **权限**：RBAC（角色-权限-用户）
- **缓存**：Redis（可选）
- **日志**：Winston / Pino
- **接口规范**：RESTful（支持后续 GraphQL）

### 3.2 备选（Java 方案）
- **框架**：Spring Boot
- **安全**：Spring Security + JWT
- **ORM**：MyBatis Plus / JPA

---

## 4. 系统结构规划

```
后台管理系统
├── 登录/注册模块
├── 用户管理模块
├── 权限管理模块
├── 菜单管理模块
└── 仪表盘首页模块
```

### 4.1 页面结构
- 左侧：菜单栏（可折叠，按权限显示）
- 右侧上部：面包屑 + 登录信息区域
- 右侧下部：具体页面内容
- 登录信息区域点击下拉：个人信息、退出登录

## 5. 任务拆分（Roadmap）

### 5.1 前端任务拆分
1. 初始化项目（Vite + React + TS）
2. 引入 UI 组件库与基础主题
3. 布局开发（侧边栏 + 头部 + 内容区）
4. 路由配置（基础路由 + 权限路由）
5. 登录/注册页面原型
6. 首页仪表盘页面原型
7. 用户管理页面（列表+分页+操作）
8. 菜单与权限管理页面
9. 登录信息下拉 + 退出逻辑
10. 接口对接与鉴权封装

### 5.2 后端任务拆分
1. 项目初始化（NestJS）
2. 数据库建模（用户/角色/权限/菜单）
3. 登录注册接口（JWT）
4. 用户管理接口
5. 权限与菜单管理接口
6. 仪表盘统计接口
7. 权限校验中间件

## 6. 接口文档（草案）

> 统一前缀：`/api/v1`

### 6.1 认证模块

#### 登录
- **POST** `/auth/login`
- 请求：
```json
{ "username": "admin", "password": "123456" }
```
- 响应：
```json
{ "token": "jwt-token", "refreshToken": "refresh", "user": { "id": 1, "name": "admin", "role": "super" } }
```

#### 注册
- **POST** `/auth/register`
- 请求：
```json
{ "username": "newuser", "password": "123456" }
```

#### 退出登录
- **POST** `/auth/logout`

---

### 6.2 用户管理

#### 获取用户列表
- **GET** `/users?page=1&size=10`

#### 创建用户
- **POST** `/users`

#### 更新用户
- **PUT** `/users/{id}`

#### 删除用户
- **DELETE** `/users/{id}`

---

### 6.3 菜单与权限

#### 获取菜单树
- **GET** `/menus/tree`

#### 创建菜单
- **POST** `/menus`

#### 获取权限列表
- **GET** `/permissions`

---

### 6.4 仪表盘统计

#### 获取首页统计数据
- **GET** `/dashboard/stats`
- 响应示例：
```json
{
  "userCount": 1234,
  "activeCount": 456,
  "orders": 789,
  "income": 123456.78
}
```

---

## 7. 数据模型（简要）

- **User**：id, username, password, roleId, status
- **Role**：id, name, permissions[]
- **Permission**：id, code, name
- **Menu**：id, name, path, parentId, permissionCode
